# Service Handover Deliverable Guide (Draft)

이 문서는 랜딩페이지 서비스 카드에서 약속 가능한 "운영 이관 산출물" 범위를 명확히 정의하기 위한 초안이다.

## 목적
- 과장된 약속 없이, 고객이 실제로 운영에 필요한 최소 정보를 받도록 기준을 통일한다.
- 제안서/상담/카피에서 동일한 문구를 사용해 기대치 불일치를 줄인다.

## 공통 원칙
- 보장하지 못하는 표현(예: 기간 미정 안정화 지원)은 사용하지 않는다.
- 결과물 범위는 계약서/제안서에 명시된 범위를 우선한다.
- 비밀값(실제 비밀번호/토큰)은 문서에 직접 기록하지 않는다.

## 카드별 결과물 기준

### 카드 1: 자동화 실행 진단 (90분)
권장 결과물 문구:
- 진단 요약 문서 + 2주 실행 후보 리스트

포함 항목:
- 현재 흐름의 핵심 병목/리스크 요약
- 우선순위 후보 작업 목록(예: 작업명, 목적, 난이도, 선행조건)
- 2주 내 시도 가능한 실행 후보 제안

주의:
- 성과 보장 문구 금지
- "2주 내 완료 보장"으로 해석될 수 있는 표현 금지

### 카드 2: 자동화 실행 구축 (2~4주)
권장 결과물 문구:
- 운영 가능한 자동화 구성 + 자동화 운영 가이드 문서

포함 항목:
- 자동화 흐름 개요
- 실행/중지 방법
- 기본 장애 점검 포인트
- 수정 지점(변수/연동 포인트)

### 카드 3: 외주용역 집중 트랙 (4~8주)
권장 결과물 문구:
- 구축 결과물 + 운영 이관 문서 (후속 지원 범위 별도 합의)

운영 이관 문서 포함 항목(권장):
- 구성 개요: 개발 스택, PaaS/인프라, 연동 구성
- 배포/실행: 배포 절차, 서버 재시작 방법, 기본 롤백 절차
- 환경설정: 필수 환경변수 목록(값 제외), 비밀정보 관리 위치
- 운영 점검: 로그 위치, 1차 장애 점검 순서, 자주 발생 이슈 대응
- 외부 의존성: 도메인/DNS, 메일, 3rd-party 연동 현황
- 운영 연락처: 플랫폼 고객센터/지원 채널, 내부 에스컬레이션 경로

주의:
- "안정화 지원" 단독 문구는 기간/범위가 불명확하므로 사용 금지
- 후속 운영 지원은 별도 계약 범위로 분리 명시

## 카피에 바로 쓰는 안전 문구
- 카드 1: 진단 요약 문서 + 2주 실행 후보 리스트
- 카드 2: 운영 가능한 자동화 구성 + 자동화 운영 가이드 문서
- 카드 3: 구축 결과물 + 운영 이관 문서 (후속 지원 범위 별도 합의)

## 향후 반영 제안 (구현 단계)
- 관리자 전용 페이지에 "산출물 기준" 섹션으로 노출
- 관리자 정보 구조는 `단일 템플릿형 + 카드별 앵커 목차`를 권장
- 제안서 템플릿/운영 체크리스트와 동일 문구 동기화
- 상담 메일 자동응답에는 결과물 범위 요약을 포함하지 않음
