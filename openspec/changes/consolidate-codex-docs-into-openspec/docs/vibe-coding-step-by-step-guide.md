# 큐룸 랜딩페이지 바이브코딩 단계별 가이드

## 0) 목표
- 목적: 명세 중심으로 Codex를 활용해 빠르게 랜딩페이지를 제작하고, 신뢰도(경력/실적/사업자 정보)를 일관되게 반영한다.
- 기준 문서: `codex-document/quroom-landing-spec.md`

---

## 1) 작업 준비
1. Cursor에서 프로젝트 루트를 `homepage`로 열기
2. Codex 연결 상태 확인(API 키/모델)
3. 기준 문서 열기:
   - `codex-document/quroom-landing-spec.md`
4. 이미지 경로 확인:
   - `images/portfolio/thumb`
   - `images/portfolio/detail`

완료 기준:
- Codex가 명세 파일을 읽고 요약할 수 있어야 함

---

## 2) 명세 동결(Freeze) 먼저 하기
1. 아래 항목이 `추후 입력` 없이 채워졌는지 확인
   - 사업자등록번호
   - 로고/OG 이미지 경로
   - 개인정보처리방침/이용약관 링크
2. 동적 연차 기준값 최종 확인
   - preCareerStart / preCareerEnd
   - personalCareerStart / personalCareerEnd
   - companyCareerStart
3. 실적 수치 고정
   - `총 7건 (개인 5, 외주 2)`

완료 기준:
- 핵심 카피/숫자/링크가 변경되지 않는 상태

---

## 3) Codex에 첫 구현 지시
아래 프롬프트를 그대로 사용:

```text
`codex-document/quroom-landing-spec.md`를 기준으로 원페이지 랜딩을 구현해줘.
조건:
1) 명세 우선, 임의 변경 금지
2) 섹션: 헤더/Hero/About/서비스/포트폴리오/문의/푸터
3) 반응형(모바일 우선)
4) SEO 메타/OG 기본 반영
5) 연차는 명세의 경력 기준값으로 동적 계산
6) 포트폴리오는 images/portfolio 파일명 규칙을 사용
작업 후 변경 파일 목록과 실행 방법을 알려줘.
```

완료 기준:
- 페이지 기본 구조와 콘텐츠가 명세 순서대로 렌더링됨

---

## 4) 2차 지시: 신뢰 요소 강화
아래 프롬프트 사용:

```text
다음 항목만 보강해줘:
1) Hero/About에 동적 경력 문구 노출
2) 푸터에 사업자 정보 요약 + 대표자 개인 SNS 라벨 반영
3) 포트폴리오 카드에 신뢰 배지(실제 구축/성과 지표/익명 처리 가능) 추가
4) 문의 섹션의 CTA/응답시간 문구 가독성 개선
명세 밖 카피 변경은 제안 후 반영해줘.
```

완료 기준:
- 신뢰 관련 문구가 Hero/About/Portfolio/Footer에서 일관됨

---

## 5) 3차 지시: 품질 마감
아래 프롬프트 사용:

```text
런칭 전 점검을 해줘.
1) 모바일/태블릿/데스크탑 반응형 점검
2) 접근성(alt, 대비, 키보드 이동) 점검
3) 성능 최적화(이미지 크기/지연 로딩) 적용
4) SEO 타이틀/메타/OG/파비콘 확인
5) 누락/오타/깨진 링크 수정
그리고 점검 결과를 체크리스트로 보고해줘.
```

완료 기준:
- 명세의 DoD 항목 체크 가능 상태

---

## 6) 작업 방식 규칙 (바이브코딩 핵심)
1. 한 번에 크게 시키지 말고, 단계별로 지시
2. 각 단계마다 "변경 파일 + 이유 + 검증 결과"를 받기
3. 카피 수정과 구조 수정을 한 프롬프트에 섞지 않기
4. 숫자/법정 문구는 명세 기준 고정
5. 애매한 값은 `추후 입력`으로 남기지 말고 TODO로 분리
