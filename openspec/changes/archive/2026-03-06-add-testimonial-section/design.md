## Context

현재 랜딩에는 후기를 수집하는 전용 흐름이 없고, 공개 페이지에서 누구나 남기는 방식은 스팸/품질 저하 위험이 크다. 이번 변경의 1차 목적은 "대면 무료 상담 직후 운영자가 개별 전달하는 링크로만 후기 작성"을 가능하게 해 즉시 운영 가능한 수집 루프를 만드는 것이다.

## Goals / Non-Goals

**Goals:**
- 초대 링크(토큰) 기반 비공개 후기 작성 페이지를 제공한다.
- 작성된 후기를 검토 대기 상태로 저장해 운영자가 승인 후 노출할 수 있게 한다.
- 승인 후기 최소 `N=3`건 기준을 충족하기 전에는 공개 후기 섹션을 노출하지 않는다.
- 내일 미팅부터 바로 사용할 수 있는 단순 운영 절차(발급 → 전달 → 작성 → 검토)를 지원한다.

**Non-Goals:**
- 후기 섹션 디자인 최적화/정렬 실험
- 후기 평점/이미지 업로드/영상 첨부 기능
- 외부 SNS 후기 자동 수집
- 복잡한 권한 시스템(역할 기반 세분화)

## Decisions

- 선택: 후기 작성 URL은 공개 네비게이션에 노출하지 않고, 토큰 링크로만 접근 허용.
  - 이유: 공개 폼 노출 시 스팸/저품질 입력 가능성이 높고, 대면 상담 후 후기를 받으려는 목적과 맞지 않다.
  - 대안: 누구나 접근 가능한 `/testimonials/new`.
    - 미채택 이유: 운영 검수 부담 증가, 품질 불일치 위험.

- 선택: 초대 링크는 기본 1회 제출 제한 + 만료 7일 정책을 사용.
  - 이유: 링크 유출/재사용 위험을 줄이면서도 상담 직후 작성 유도에 충분한 기간이다.
  - 대안: 무기한/무제한 링크.
    - 미채택 이유: 후기 중복/대리작성 가능성 증가.

- 선택: 제출 후 상태는 `pending`(검토 대기)로 저장하고 기본 비공개 처리.
  - 이유: 대면 후기에 포함될 수 있는 민감 정보/과장 표현을 운영자가 확인할 수 있어야 한다.
  - 대안: 제출 즉시 자동 공개.
    - 미채택 이유: 브랜드 리스크 및 법적 리스크 증가.

- 선택: 공개 후기 섹션 노출 기준은 "승인 후기 3건 이상"으로 시작.
  - 이유: 신뢰 신호 최소치 확보와 운영 시작 속도의 균형점.
  - 대안: 1건부터 노출 또는 5건 이상 고정.
    - 미채택 이유: 1건은 신뢰감 약함, 5건은 초기 노출 지연 가능성 큼.

- 선택: 운영자가 링크를 수동 발급/전달하는 단순 워크플로우를 1차 채택.
  - 이유: 내일 미팅 즉시 적용 가능하며 학습 비용이 낮다.
  - 대안: 상담 완료 트리거 자동 발급/메시지 자동 발송.
    - 미채택 이유: 초기 구현 범위가 커지고 외부 연동이 필요.

## Risks / Trade-offs

- [링크 공유로 제3자 제출 가능] → 만료시간/1회 제한/관리자 검토로 완화
- [운영 수동 작업 증가] → 링크 발급 템플릿 문구 표준화, 추후 자동화 change로 분리
- [초기 후기 수 부족으로 섹션 미노출 장기화] → 공개 기준 `N`을 운영 설정으로 조정 가능하게 설계
- [실명 후기 부담으로 작성률 저하] → 표시명 옵션(이니셜/업종) 허용 검토

## Migration Plan

1. 후기 및 초대 링크 저장 구조 추가(상태/만료/사용 여부 포함).
2. 비공개 후기 작성 라우트와 토큰 검증 흐름 추가.
3. 관리자 검토/승인/비공개 전환 작업 경로 추가.
4. 랜딩 공개 후기 섹션을 승인 건수 기준으로 조건부 노출.
5. 운영 체크: 실제 상담 직후 링크 전달 -> 작성 완료 -> 관리자 승인까지 E2E 점검.
6. 롤백 시: 공개 노출만 비활성화하고 수집 데이터는 보존.
