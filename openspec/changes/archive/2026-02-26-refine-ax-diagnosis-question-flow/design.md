## Context

현재 무료 AX 진단은 8개 문항 기반으로 빠른 점수 산출에는 유리하지만, 업무 맥락과 문제 유형을 정교하게 분리하지 않아 결과 개인화 한계가 있다. 이번 변경은 진단 항목을 다축으로 재구성해 사용자 상황을 더 정확히 분류하고, 결과 화면/메일에서 바로 실행 가능한 우선순위와 도구 제안을 제공하는 것이 목적이다.

영향 범위는 `landing/forms.py`, `landing/views.py`, `landing/ax_tool_stack.py`, `landing/mailers.py`, 진단 관련 템플릿과 테스트로, 프론트/백엔드/콘텐츠가 함께 바뀌는 교차 모듈 변경이다.

## Goals / Non-Goals

**Goals:**
- 다축 진단(최소 5축) 입력 구조로 설문 정확도를 높인다.
- 점수(A/B/C) 외에 운영 유형/병목 유형/실행 준비도 유형을 함께 산출한다.
- 결과 메시지와 이메일 리포트를 체크 패턴 기반으로 개인화한다.
- 기존 문의 저장/메일 발송 안정성은 유지한다.

**Non-Goals:**
- 관리자 대시보드의 대규모 UI 재설계
- 외부 분석 플랫폼(GA4 등) 신규 연동
- 결제/예약 기능 추가

## Decisions

### Decision 1: 문항을 축 기반 그룹으로 정의한다
- 선택: 문항을 단일 리스트가 아닌 `업무 맥락/반복 병목/데이터 상태/자동화 적합도/실행 준비도` 그룹으로 정의
- 이유: 그룹별 서브 스코어를 계산해 결과를 단순 총점보다 해석 가능하게 만들기 위함
- 대안:
  - 단일 총점 유지: 구현은 쉽지만 제안 근거가 약함
  - 완전 자유 서술형: 정성 정보는 좋으나 자동 분류와 테스트가 어려움

### Decision 2: 결과 엔진을 규칙 기반으로 유지한다
- 선택: 초기 버전은 규칙 기반(체크 조합, 축별 임계치)으로 결과 분류
- 이유: 설명 가능성, 테스트 용이성, 운영 안정성이 높음
- 대안:
  - LLM 실시간 생성: 문장 품질은 높을 수 있으나 일관성/재현성 관리 비용 증가

### Decision 3: 도구 추천은 경험 기반 세트 + 선택 확장으로 제공한다
- 선택: 사용 경험이 있는 도구를 기본 추천으로, 필요 시 대체 옵션을 함께 노출
- 이유: 실행 가능성이 높은 추천을 우선 보장
- 대안:
  - 시장 전체 도구 무작위 추천: 과다 선택 유도, 실제 실행률 저하 가능

### Decision 4: 기존 A/B/C 등급은 유지하고 세부 분류를 추가한다
- 선택: 기존 등급 체계를 유지해 사용자 혼란을 줄이고, 내부적으로 세분화 결과를 추가
- 이유: 기존 UX/카피를 크게 깨지 않고 정밀도만 향상 가능
- 대안:
  - 등급 체계 전면 변경: 커뮤니케이션 비용 증가

## Risks / Trade-offs

- [Risk] 문항 수 증가로 이탈률이 올라갈 수 있음
  - Mitigation: 축별 핵심 문항만 유지하고, 진행 안내/소요시간 표기를 명확화
- [Risk] 규칙 기반 분류가 과도하게 단순화될 수 있음
  - Mitigation: 축별 근거 문구를 함께 노출하고, 추후 운영 데이터로 규칙 보정
- [Risk] 템플릿/뷰/메일 동시 변경으로 회귀 가능성 증가
  - Mitigation: 기존 테스트 보강 + 진단 제출/결과/메일 렌더링 테스트 추가

## Migration Plan

1. 축 기반 질문 정의를 `landing/ax_tool_stack.py`에 추가
2. 폼 필드와 검증 규칙을 `landing/forms.py`에 반영
3. 진단 산출 로직/결과 컨텍스트를 `landing/views.py`에 반영
4. 결과 화면 및 리포트 템플릿 업데이트
5. 테스트 업데이트 후 `./scripts/verify.sh` 실행
6. 문제 발생 시 기존 질문 매핑과 결과 생성 함수를 롤백
