## ADDED Requirements

### Requirement: Main services section MUST be segmented into three parts
The system MUST present founder services in three explicit groups: AX execution, outsourcing-focused track, and startup foundation infrastructure.

#### Scenario: Three-part service structure is rendered
- **WHEN** a user views the main landing services section
- **THEN** service content is grouped into AX execution, outsourcing-focused track, and startup foundation infrastructure
- **AND** each group has a distinct heading and explanatory copy

### Requirement: Outsourcing-focused track MUST use the same card layout as AX cards
The system MUST render outsourcing-focused offerings with the same information hierarchy and visual card pattern used by AX offerings.

#### Scenario: Consistent card layout across founder core offers
- **WHEN** a user compares AX cards and outsourcing-focused cards
- **THEN** both use the same card layout pattern for summary, scope items, duration/price/audience, and CTA
