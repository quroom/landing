## Context

현재 무료 AX 진단 결과는 `우선 실행 1개` 중심으로 간결하지만, 실제 설문 문항(8개)과 `tools_map`(8 intent) 대비 내부 패턴 커버리지가 충분히 보장되지 않아 intent 누락 위험이 있다.

또한 채널(웹 결과, preview, 이메일) 간 결과 계약이 완전히 고정되어 있지 않아, 동일 응답에서도 노출 구성 차이가 발생할 수 있다. 이번 변경은 사용자 노출은 기존처럼 "우선순위 1개"를 유지하고, 내부 패턴/검증 기준에서 8 intent 누락이 없도록 보장하는 것이 핵심이다.

## Goals / Non-Goals

**Goals:**
- 심각도 기반 응답 패턴을 최소 8종으로 확장한다.
- 8개 action intent(`action_titles`/`tools_map`)가 결과 집합에서 누락되지 않도록 규칙을 설계한다.
- 사용자 노출 결과 구조를 `우선 실행 1개`로 고정한다.
- 웹/preview/이메일에서 동일한 결과 계약(동일 입력 -> 동일 섹션 의미)을 사용한다.
- 테스트에서 `8종 패턴 존재`와 `8 intent 커버리지`를 검증한다.

**Non-Goals:**
- 설문 문항 수 자체를 변경하지 않는다.
- 신규 외부 툴 연동이나 추천 툴 DB를 도입하지 않는다.
- 카피 전면 재브랜딩(톤앤매너 전체 교체)은 범위에서 제외한다.

## Decisions

### 1) 결과 계산을 2단계로 분리한다 (심각도 평가 -> 결과 조립)
- 선택: 현재 로직을 "점수/취약도 평가"와 "최종 응답 조립"으로 분리한다.
- 이유: 우선 실행 1개 선정과 후속 후보 노출 규칙을 독립적으로 관리해야 8종 패턴 확장이 쉬워진다.
- 대안: 단일 함수에서 분기 계속 확장.
  - 미채택 이유: 분기 누적 시 preview/이메일 불일치와 회귀 위험이 커진다.

### 2) 응답 패턴은 "대표 intent + 보조 intent 묶음"으로 정의한다
- 선택: 패턴을 intent 조합 단위로 정의하고, 각 패턴에 고정 카피/툴 노출 규칙을 부여한다.
- 이유: 최소 8종을 명시적으로 관리할 수 있고, QA에서 패턴 커버리지를 측정하기 쉽다.
- 대안: 실시간 점수 상위 N개를 즉석 조합.
  - 미채택 이유: 패턴 수는 늘어도 메시지 일관성이 낮아지고 검증 포인트가 불명확해진다.

### 3) 8 intent 커버리지는 "패턴/검증 집합 기준"으로 보장한다
- 선택: 단일 사용자 응답은 우선 실행 1개만 노출하고, 전체 패턴 집합/검증 시나리오에서 8 intent가 모두 나타나도록 보장한다.
- 이유: 사용자 결과의 간결성을 유지하면서도 추천 축 누락을 구조적으로 방지할 수 있다.
- 대안: 사용자별 결과에 8개 intent 전부 노출.
  - 미채택 이유: 결과가 과도하게 길어져 핵심 행동 유도(우선 실행 1개)가 약해진다.

### 4) 채널 공통 섹션 빌더를 단일 소스로 유지한다
- 선택: preview/이메일/웹 모두 공통 섹션 생성 결과를 재사용한다.
- 이유: "preview = 메일" 일치 조건을 유지하면서 카피 드리프트를 방지한다.
- 대안: 채널별 템플릿에서 독립 가공.
  - 미채택 이유: 과거 동일 이슈(불일치)가 반복될 가능성이 높다.

### 5) 테스트는 "패턴 수"와 "intent 커버리지"를 분리 검증한다
- 선택: (a) 최소 8종 패턴 존재, (b) 패턴 전체에서 8 intent 커버, (c) 채널별 렌더 일치 검증을 각각 테스트한다.
- 이유: 실패 원인을 즉시 파악 가능하고 회귀 시 수정 범위를 좁힐 수 있다.
- 대안: 통합 스냅샷 1개만 유지.
  - 미채택 이유: 실패시 원인 식별이 어렵고 유지보수 비용이 증가한다.

## Risks / Trade-offs

- [패턴 확장으로 카피 중복 증가] -> 공통 문장 템플릿 + 변수 치환 구조로 중복을 제한한다.
- [우선 실행 1개만 노출 시 정보 부족 체감] -> preview/관리자 검증 경로에서 intent 커버리지 상태를 별도로 확인 가능하게 유지한다.
- [테스트 케이스 증가로 유지비 상승] -> intent/패턴 매트릭스 기반 파라미터 테스트로 압축한다.
- [운영자가 8 intent 누락 여부를 놓칠 위험] -> 패턴 매트릭스 테스트와 preview 점검 시나리오를 고정해 사전 검수한다.

## Migration Plan

1. 결과 조립 로직을 공통 함수로 정리하고 기존 템플릿 호출부를 해당 함수 기반으로 전환한다.
2. 8종 패턴 정의(카피/툴/우선 실행 규칙)를 데이터 구조로 선언한다.
3. preview/이메일/웹 렌더 경로를 공통 섹션 빌더 기반으로 정렬한다.
4. 테스트에 패턴 수/intent 커버리지/채널 일치 검증을 추가한다.
5. 기존 대표 시나리오(만점 포함) 회귀 검증 후 배포한다.

Rollback:
- 신규 패턴 매핑 사용 플래그를 되돌려 기존 우선 실행 1개 로직만 사용하도록 복귀 가능하게 유지한다.

## Resolved Decisions

- 8종 패턴 기준은 향후 산업군/규모 축을 추가하지 않고, 현재는 `intent` 기준으로만 유지한다.
- 사용자 노출은 우선 실행 1개로 고정하고, preview 커버리지는 우선 실행 기준으로 묶되 제목이 달라도 본문이 동일하면 하나로 그룹화한다.
- preview는 관리자 검토 목적 채널로 간주하고, 실제 메일 형식과의 UI/표현 차이는 크게 허용한다.
