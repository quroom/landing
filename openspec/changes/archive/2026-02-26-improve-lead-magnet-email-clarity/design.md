## Context

무료 자동화 실행 진단은 창업가를 대상으로 운영한다. 현재 문제는 메일이 길고 어렵게 느껴져 읽고 바로 행동하기 어렵다는 점이다. 또한 폼 제출 후 화면 리포트와 실제 메일 포맷이 다르면 사용자 경험이 분리된다.

이번 변경은 다음을 동시에 맞춘다.
- 창업가 중심의 쉬운 문장과 짧은 구조
- 메일과 미리보기(Preview) 섹션 구조 동기화
- CTA 링크를 홈페이지 문의 섹션(`/#contact`)으로 고정
- 발신자 표시명을 `큐룸`으로 고정

## Goals / Non-Goals

**Goals:**
- 메일 본문을 `진단 요약 -> 핵심 취약 카테고리 1개 -> 2주 실행 우선 1개 -> 다음 액션` 구조로 고정한다.
- 어려운 용어를 줄이고 창업가가 바로 이해할 수 있는 문장으로 통일한다.
- 메일 CTA가 항상 홈페이지 문의 섹션으로 이동하도록 한다.
- 발신자 표시명을 `큐룸 <help@quroom.kr>`으로 유지한다.
- 메일 변경 시 Preview도 같은 구조를 반영한다.
- 전체 카테고리 4개는 기본 노출하지 않고, 선택적으로 확인 가능한 보조 정보로 전환한다.

**Non-Goals:**
- 진단 점수 계산 로직 자체 변경
- 설문 문항 수/축 구조 변경
- 외국인 개발자 전용 페이지 메시지 재설계

## Decisions

### 1) 메일 본문 생성은 서버 단일 함수에서 관리
- 선택: `landing/mailers.py`에 사용자 수신 메일 전용 빌더 함수를 두고, 텍스트/HTML를 같은 데이터에서 생성한다.
- 이유: 포맷 변경 시 텍스트/HTML 드리프트를 줄이고 테스트 지점을 단일화할 수 있다.
- 대안: 템플릿 파일로 완전 분리.
  - 미채택 이유: 현재는 분기 로직(등급/약한 축/CTA)이 파이썬 함수에 이미 모여 있어, 단계적으로 정리하는 것이 안전하다.

### 1-1) 카테고리 노출 전략은 `1개 기본 + 전체 선택`
- 선택: 기본 메일/Preview에서는 핵심 취약 카테고리 1개만 보여준다.
- 이유: 창업가 관점에서 첫 행동을 빠르게 결정하도록 돕기 위해서다.
- 대안: 4개 카테고리 전체를 항상 노출.
  - 미채택 이유: 메시지가 길어지고 행동 유도(CTA)가 약해진다.

### 2) CTA 링크는 앵커가 오면 절대경로로 정규화
- 선택: `#contact` 입력 시 메일/Preview 모두 `https://<base>/#contact`로 변환한다.
- 이유: 수신 메일 클라이언트/외부 페이지에서 앵커만 전달되면 동작이 불명확하다.
- 대안: 상대 링크(`/#contact`) 유지.
  - 미채택 이유: 수신 환경에 따라 base URL이 보장되지 않는다.

### 3) 문의 유형은 창업가 서비스 구조와 일치
- 선택: 홈 문의 유형을 `자동화 실행 진단/구축/창업 기본 인프라 구축/외주용역 집중 트랙`으로 정렬한다.
- 이유: 랜딩 서비스 카드와 문의 옵션이 같아야 전환 단계에서 혼선이 없다.
- 대안: 기존 `서비스 개발` 등 범용 라벨 유지.
  - 미채택 이유: 실제 제공 패키지와 불일치해 문의 분류 품질이 떨어진다.

### 4) 발신자 이름은 환경변수로 관리
- 선택: `.env`/`.env.sample`의 `DJANGO_DEFAULT_FROM_EMAIL`에 `큐룸 <help@quroom.kr>`를 사용한다.
- 이유: 코드 수정 없이 발신자 표시명을 바꿀 수 있고 운영 환경 반영이 쉽다.
- 대안: 코드 내 하드코딩.
  - 미채택 이유: 환경별 설정 분리가 깨진다.

## Risks / Trade-offs

- [기본 노출 축소로 상세 정보 부족 체감 가능] → 전체 4개 카테고리는 선택적으로 확인 가능한 상세 섹션으로 제공한다.
- [클라이언트별 발신자명 인코딩 차이] → 기본값은 `큐룸`, 표시 이슈 발생 시 `QuRoom`으로 즉시 전환 가능하게 유지한다.
- [Preview와 실제 메일 포맷 불일치 재발] → 변경 시 테스트에서 CTA 링크/섹션 제목을 함께 검증한다.

## Migration Plan

1. `landing/mailers.py`에서 사용자 수신 메일 포맷/링크 정규화 로직 반영
2. `landing/views.py` 및 Preview 렌더 데이터 동기화
3. 홈 문의 유형 라벨/필터/표시 라벨 동기화
4. `.env.sample` 기준값 업데이트
5. 테스트 실행(`landing.tests.test_contact_form`, `landing.tests.test_landing_pages`)

롤백은 메일 빌더 함수 변경 전 커밋으로 되돌리면 된다.
