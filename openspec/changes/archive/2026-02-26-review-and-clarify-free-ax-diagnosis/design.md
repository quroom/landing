## Context

현재 무료 진단은 이미 리포트/메일/프리뷰를 제공하고 있지만, 사용자가 즉시 행동하기 어려운 문제가 남아있다. 특히 진단 축이 많고 결과 텍스트가 장문이어서 "우선 2주에 무엇을 할지"가 흐려진다.

## Goals / Non-Goals

**Goals:**
- 진단 카테고리를 4개 축으로 통일해 해석을 단순화한다.
- 각 카테고리에 대해 A/B/C 등급별 문구를 명시적으로 제공한다.
- 실행 제안은 1개 과제만 제공해 의사결정 부담을 줄인다.
- 툴 추천은 핵심 툴만 노출해 실무 적용 가능성을 높인다.
- 결과 카드/프리뷰/이메일에서 동일한 메시지 구조를 유지한다.

**Non-Goals:**
- 진단 문항 수(10문항) 자체 변경
- 새로운 외부 분석도구 도입
- 관리자 대시보드 기능 확장

## Decisions

### Decision 1: 카테고리 구조를 4개로 고정

- `workflow_clarity`, `data_operation_base`, `automation_design`, `execution_system` 4개 축으로 통일한다.
- 이유: 진단 해석의 인지 부하를 줄이고 설명 가능성을 높이기 위함.

### Decision 2: 카테고리별 A/B/C 메시지 테이블 사용

- 축별 점수로 축 등급을 계산하고, `(카테고리, 등급)` 매핑표에서 메시지를 선택한다.
- 이유: 총점 기반 단일 메시지보다 진단 신뢰도와 개인화 체감이 높다.

### Decision 3: 2주 실행 제안은 1개만 제공

- 최저 점수 문항 1개를 우선 과제로 선택하고 실행 기준을 함께 제시한다.
- 이유: 실행 전환이 목표인 랜딩 문맥에서 다수 과제 제시는 오히려 이탈을 유발한다.

### Decision 4: 툴 추천은 소수 정예

- 우선 과제 기준 상위 툴 2개, 전체 추천은 최대 3개로 제한한다.
- 이유: 과도한 툴 목록은 불확실성을 높이고 실제 실행률을 떨어뜨린다.

### Decision 5: 메일은 등급별 안내 + 단일 CTA

- 메일에 등급별 안내 문구를 추가하고 CTA는 1개만 유지한다.
- 이유: 클릭 분산을 막고 상담 전환 경로를 명확히 한다.

## Risks / Trade-offs

- 정보 축약으로 일부 사용자가 "디테일 부족"을 느낄 수 있음
  - 대응: 상세가 필요하면 상담 CTA에서 확장 안내
- 최저 점수 1개만 추천 시 다른 문제를 놓칠 수 있음
  - 대응: 리포트에 카테고리 4개 결과를 병행 표기해 전체 상황은 유지
