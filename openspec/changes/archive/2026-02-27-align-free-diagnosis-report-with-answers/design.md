## Context

카테고리 노출 제거 변경 이후에도 `free-diagnosis/preview/`에는 여전히 4개 분류형 상세 블록이 남아 있고, 핵심 보완 포인트 보조 문장에는 `우선 항목`이 노출되고 있다.  
또한 2주 실행 과제의 완료 기준 문구가 intent별로 유사한 접두 문장을 반복해 검토와 실행 지시력이 떨어진다.  
이번 변경은 점수 계산 모델을 바꾸지 않고, 결과 표현 계층(웹/preview/이메일)에서 응답-결과 매핑을 간결하게 재정렬하는 것이 핵심이다.

## Goals / Non-Goals

**Goals:**
- preview에서 4개 분류형 보완 포인트 상세 노출을 제거하고 핵심 보완 포인트 1개 중심으로 단순화한다.
- 핵심 보완 포인트 보조 문장에서 `우선 항목: ...` 문구를 제거해 불필요한 내부 문항 노출을 없앤다.
- 2주 실행 과제 완료 기준 문구를 intent별로 중복 없이 재작성해 실행 기준을 더 직접적으로 전달한다.
- 제목 외 본문이 동일한 preview 시나리오는 묶어서 보여 검토 효율을 높인다.
- 웹/preview/이메일 공통 섹션 계약과 회귀 테스트를 유지한다.

**Non-Goals:**
- 문항 개수(8개) 또는 필수/선택 구조 변경
- 우선순위 점수식 `(2 - answer_score) * impact_weight` 변경
- 문의 접수/메일 전송 파이프라인 구조 변경

## Decisions

### 1) 4개 분류형 상세는 내부 계산용으로 유지하고 preview 노출에서는 제외한다
- 선택: `category_insights` 계산은 유지하되 preview 템플릿/컨텍스트에서는 핵심 보완 포인트 단일 블록만 렌더링한다.
- 이유: 내부 회귀/확장 여지를 남기면서 사용자 검토 화면을 단순화할 수 있다.
- 대안: `category_insights` 자체를 제거.
  - 미채택 이유: 다른 채널/디버깅/회귀 테스트에서 참조 가능한 내부 구조까지 동시에 줄이면 변경 범위가 커진다.

### 2) 핵심 보완 포인트 보조 문장에서 anchor 노출을 제거한다
- 선택: `message_secondary` 생성 시 `우선 항목: ...` 접미 구문을 제거하고 축별 행동 가이드 문장만 노출한다.
- 이유: 사용자 노출 문구가 질문 원문 재노출 없이도 충분히 이해 가능하고, 카테고리 제거 방향과 일관된다.
- 대안: anchor 문구를 유지하되 표현만 순화.
  - 미채택 이유: 불필요한 정보량과 표현 중복 문제가 그대로 남는다.

### 3) 완료 기준 문구는 intent별 단문 기준으로 재작성한다
- 선택: `one_action.execution`을 공통 접두 문장 없이 intent별 완료 조건 단문으로 정의한다.
- 이유: 중복 텍스트를 줄이고, 항목 간 차이를 즉시 읽을 수 있다.
- 대안: 기존 2줄 구조 유지 + 예시만 일부 수정.
  - 미채택 이유: 동일한 접두가 반복되어 검토 피로가 지속된다.

### 4) preview는 제목 제외 동일 본문을 그룹화해 보여준다
- 선택: preview 데이터 생성 단계에서 제목을 제외한 본문 시그니처(핵심 섹션/CTA/리포트 텍스트)로 그룹화하고, 그룹 내 시나리오 제목 목록만 병합해 노출한다.
- 이유: 중복 카드 제거로 검토 속도를 높이고, 실제로 다른 응답인지 아닌지 판단을 명확히 할 수 있다.
- 대안: 템플릿에서 단순 `ifchanged` 처리.
  - 미채택 이유: 템플릿 수준 비교는 비교 기준이 불명확하고 테스트 고정이 어렵다.

## Risks / Trade-offs

- [그룹화 기준이 과도하게 넓으면 다른 케이스가 합쳐질 수 있음] -> 시그니처에 핵심 섹션 텍스트와 CTA를 포함하고 테스트로 고정한다.
- [상세 4항목 비노출로 내부 근거를 덜 보여줄 수 있음] -> 핵심 보완 포인트/점수/등급/진단유형은 유지해 최소 설명력을 보장한다.
- [카피 정리로 기존 스냅샷 대량 변경 가능] -> `landing_pages`, `lead_magnet_sections`, `contact_form` 회귀 테스트를 함께 갱신한다.

## Migration Plan

1. 결과 생성 로직에서 `message_secondary`와 `one_action.execution` 문구 계약을 갱신한다.
2. preview 컨텍스트에서 제목 제외 동일 본문 그룹화 로직을 추가하고 템플릿 렌더링을 단순화한다.
3. preview 템플릿의 4개 분류형 상세 블록을 제거한다.
4. 웹/preview/이메일 테스트 스냅샷과 노출 문자열 검증을 갱신한다.

## Open Questions
