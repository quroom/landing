# 초보자를 위한 일반 바이브코딩 가이드

## 1) 바이브코딩이란?
- AI에게 "전체를 한 번에" 맡기는 게 아니라,
- 작은 단위로 지시하고, 결과를 확인하며, 다음 지시를 주는 작업 방식입니다.

핵심은 `짧은 반복`입니다.
- 지시 -> 결과 확인 -> 수정 지시 -> 다시 확인

---

## 2) 시작 전에 준비할 것
1. 목표 1줄
- 예: "회사 소개 랜딩페이지를 만든다."

2. 필수 정보 모으기
- 회사/서비스 소개 문구
- 연락처/링크
- 이미지/로고 파일

3. 완료 기준 정하기
- 예: "모바일에서 깨지지 않고, 문의 버튼이 동작하면 완료"

---

## 3) 초보자용 작업 순서
1. 명세(요구사항) 먼저 작성
- 페이지에 어떤 섹션이 필요한지 목록화
- 카피(문구)를 먼저 확정

2. 1차 구현 지시
- "레이아웃만 먼저" 요청
- 디자인보다 구조를 먼저 만듦

3. 2차 구현 지시
- 텍스트/이미지/링크 반영
- 반응형(모바일) 수정

4. 3차 구현 지시
- 접근성/SEO/성능 점검

5. 최종 점검
- 오타, 깨진 링크, 버튼 동작 확인

---

## 4) 좋은 프롬프트 공식
아래 5가지를 포함하면 정확도가 올라갑니다.

1. 무엇을 만들지
2. 어떤 파일을 기준으로 할지
3. 무엇을 절대 바꾸면 안 되는지
4. 결과물 형식(코드, 실행법, 변경 파일 목록)
5. 검증 기준

예시:

```text
다음 명세 파일을 기준으로 원페이지를 구현해줘: docs/spec.md
조건:
1) 섹션 순서 고정(헤더/Hero/서비스/문의/푸터)
2) 모바일 우선 반응형
3) 임의 카피 수정 금지
4) 작업 후 변경 파일과 실행 방법을 알려줘
```

---

## 5) 초보자가 자주 하는 실수
1. 한 번에 너무 많이 시킴
- 해결: 한 번에 1~2개 작업만 지시

2. 기준 문서 없이 시작함
- 해결: 최소한의 명세 파일부터 작성

3. 결과 검증 없이 다음 단계로 감
- 해결: 매 단계마다 체크리스트로 확인

4. "예쁘게 해줘"처럼 모호하게 지시
- 해결: 구체적으로 지시(색상, 톤, 섹션, 우선순위)

---

## 6) 추천 체크리스트
- [ ] 텍스트/링크/연락처가 최신 정보인가?
- [ ] 모바일에서 레이아웃이 깨지지 않는가?
- [ ] 버튼/폼이 실제 동작하는가?
- [ ] 로딩 속도와 이미지 크기가 적절한가?
- [ ] SEO 기본값(title/description/OG)이 들어갔는가?

---

## 7) 초보자용 운영 팁
1. "완벽한 첫 결과"를 기대하지 말고 3회 반복을 기본으로 생각하세요.
2. 큰 수정은 "이유 + 변경 범위"를 먼저 물어본 뒤 진행하세요.
3. 중요한 문구/수치는 명세에서 잠그고(고정), 디자인만 반복 수정하세요.

---

## 8) 빠른 실행 템플릿
```text
이제부터 바이브코딩 방식으로 작업해줘.
목표: [여기에 목표]
기준 파일: [명세 파일 경로]
이번 단계 작업:
1) [작업1]
2) [작업2]
제약:
- 임의 카피 변경 금지
- 작업 후 변경 파일/검증 결과 보고
```

이 템플릿으로 단계별로 반복하면 초보자도 안정적으로 결과를 만들 수 있습니다.
