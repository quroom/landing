# 바이브코딩 프롬프트 모음 (Markdown 입력용)

초보자가 바로 복붙해서 쓸 수 있는 템플릿입니다.  
`[대괄호]` 부분만 바꿔서 사용하세요.

---

## 1) 명세서 초안 만들기

```text
아래 정보를 바탕으로 랜딩페이지 명세서 초안을 markdown으로 작성해줘.

[서비스/회사 소개]
[타겟 고객]
[핵심 제공 가치]
[연락처/SNS]

조건:
1) 원페이지 기준
2) 섹션: 헤더/Hero/About/서비스/포트폴리오/문의/푸터
3) SEO/성능/접근성 체크리스트 포함
4) "추후 입력" 항목은 TODO로 분리
```

---

## 2) 명세서 보강하기

```text
`[명세 파일 경로]`를 읽고 다음 항목을 보강해줘.
1) 신뢰 요소(경력, 실적, 링크) 문구 구체화
2) 포트폴리오 공개 원칙(실명/익명) 명시
3) 사업자 정보 노출 수준 정리
4) 중복/모순 문구 정리

수정 후:
- 변경된 항목 목록
- 수정 이유
- 남은 TODO
를 함께 보여줘.
```

---

## 3) 1차 구현(구조 중심)

```text
`[명세 파일 경로]` 기준으로 1차 구현해줘.

이번 단계 목표:
- 구조/레이아웃만 구현
- 카피는 명세 그대로 사용

조건:
1) 섹션 순서 고정
2) 모바일 우선 반응형
3) 임의 디자인 추가 최소화

작업 후:
- 변경 파일 목록
- 실행 방법
- 확인 포인트
를 알려줘.
```

---

## 4) 2차 구현(콘텐츠/신뢰 강화)

```text
기존 코드에서 아래만 보강해줘.
1) 신뢰 문구(Hero/About/Footer) 일관성 맞추기
2) 포트폴리오 카드에 결과/배지 추가
3) 대표자 개인 SNS 라벨 명확히 표기
4) 문의 섹션 CTA 가독성 개선

주의:
- 명세 밖 카피 변경은 제안 먼저
- 수치(경력/프로젝트 수)는 임의 변경 금지
```

---

## 5) 동적 연차 계산 적용

```text
연차를 고정 텍스트가 아니라 동적으로 계산해 반영해줘.

기준값:
- preCareerStart: [YYYY-MM-DD]
- preCareerEnd: [YYYY-MM-DD]
- personalCareerStart: [YYYY-MM-DD]
- personalCareerEnd: [YYYY-MM-DD]
- companyCareerStart: [YYYY-MM-DD]

요구사항:
1) 오늘 날짜 기준 총 경력 계산
2) Hero/About에 `n년 m개월` 표기
3) 계산 로직을 유틸 함수로 분리
4) 테스트 가능한 형태로 작성
```

---

## 6) 이미지 반영(파일명 규칙 기반)

```text
`images/portfolio` 폴더 파일명을 스캔해서 포트폴리오 데이터를 연결해줘.

규칙:
- thumb 이미지는 목록 카드
- detail 이미지는 상세/모달
- 파일명 패턴: YYYY-client-project-type-v1.ext

결과:
- 매핑된 카드 목록
- 누락된 파일/이상한 파일명 리포트
```

---

## 7) 최종 점검

```text
배포 전 점검을 진행해줘.
1) 반응형(모바일/태블릿/데스크탑)
2) 접근성(alt, 대비, 키보드 이동)
3) SEO(title, meta, OG, favicon)
4) 성능(이미지 최적화, lazy loading)
5) 링크/버튼/폼 동작

결과를 체크리스트 형식으로 보고하고,
문제가 있으면 파일별 수정 제안까지 작성해줘.
```

## 8) 코드리뷰 요청 프롬프트

```text
이번 변경사항을 코드리뷰해줘.

중점:
1) 버그/회귀 가능성
2) 모바일 레이아웃 깨짐 가능성
3) 접근성/SEO 누락
4) 명세 불일치

출력 형식:
- 심각도 순으로 이슈 목록
- 파일 경로 + 라인 기준
- 수정 제안
```

## 9) 초보자용 한 줄 운영 규칙
- 한 번에 1~2개만 시킨다
- 매 단계마다 검증 결과를 받는다
- 수치/법정 문구는 명세에서 고정한다
- 애매한 건 "추후 입력"이 아니라 TODO로 분리한다

## 10) 피드백 루프 프롬프트 (서로 보며 문서 업데이트)

```text
`[대상 md 파일 경로]`를 기준으로 피드백 루프를 1회 수행해줘.

이번 라운드 입력:
- 유지할 것: [문구/구조]
- 바꿀 것: [문구/구조]
- 추가할 것: [항목]
- 제거할 것: [항목]

작업 규칙:
1) 내가 말한 범위만 수정
2) 수정 이유를 항목별로 1줄씩 설명
3) 수정 후 "변경 요약 + 남은 이슈 + 다음 라운드 제안 3개"를 작성
```

---

## 11) 변경 이력 남기기 프롬프트

```text
방금 수정한 내용을 `[대상 md 파일 경로]` 하단에 "변경 이력" 섹션으로 추가해줘.

형식:
- 날짜:
- 라운드:
- 변경 요약:
- 결정 사항:
- 보류 사항:
- 다음 액션:

주의:
- 기존 본문은 유지
- 변경 이력은 최신순으로 맨 위에 추가
```
