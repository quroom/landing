## Context

현재 무료 진단은 내부 축(카테고리) 구조를 그대로 UI에 노출하고 8문항을 한 번에 제시한다. 이 방식은 문항 의도보다 구조 설명이 먼저 보이고, 카테고리 내 문항 결이 균일하지 않아 결과 시뮬레이션 메시지와 체감 매칭이 어긋나는 문제가 있다.  
또한 `하고 있지 않음` 응답이 많은 사용자에게도 우선순위 제안이 사업 임팩트 기준으로 일관되게 정렬되지 않아, “무엇부터 할지” 안내 품질이 흔들린다.

## Goals / Non-Goals

**Goals:**
- 설문 UI에서 카테고리 노출을 제거하고 질문 중심 흐름으로 단순화한다.
- 8문항 구조는 유지하되 `핵심 4문항 우선 + 추가 4문항 선택`으로 응답 부담을 낮춘다.
- `미실행 항목 비중 + 사업 임팩트` 기준으로 우선 과제 1개 추천 규칙을 명확히 한다.
- 결과 시뮬레이션/이메일 카피를 문항 의도와 일관되게 동기화한다.

**Non-Goals:**
- 진단 시스템을 새로운 통계/ML 모델로 재구축
- 문의 접수/메일 발송 파이프라인 구조 변경
- 대규모 이벤트 수집 체계(실험 플랫폼) 신규 도입

## Decisions

### 1) 설문 표시 구조와 내부 점수 축을 분리한다
- 선택: 사용자 화면은 질문만 순차 표시하고, `DIAGNOSIS_AXES`는 내부 계산/결과 구성에서만 사용한다.
- 이유: 사용자 인지 부하는 줄이고, 내부 해석 구조는 유지해 회귀 리스크를 낮출 수 있다.
- 대안: 카테고리 UI 유지 + 설명 문구만 단순화.
  - 미채택 이유: 문항 자체보다 구조가 먼저 보이는 문제를 해결하지 못한다.

### 2) 4+4 단계형 응답 모델을 채택한다
- 선택: 핵심 4문항은 `q1,q3,q5,q7`으로 고정하고 필수 응답으로 둔다. 추가 4문항(`q2,q4,q6,q8`)은 화면에 항상 노출하되 선택 응답으로 제공한다.
- 이유: 핵심 진단 일관성을 유지하면서도, 사용자에게 전체 질문 맥락을 투명하게 보여주고 UI 조건 분기를 줄일 수 있다.
- 대안: 4문항으로 완전 축소.
  - 미채택 이유: 전환은 좋아질 수 있으나 진단 해상도/후속 메시지 품질이 떨어질 가능성이 크다.

### 3) 점수/등급은 응답한 문항 기준으로 정규화한다
- 선택: 총점/축점수의 분모를 “응답한 문항 수” 기준으로 계산하고, 결과에 `간단 진단(4)`/`정밀 진단(8)` 상태를 노출한다.
- 이유: 선택 문항 미응답을 0점으로 간주할 때 생기는 왜곡을 방지한다.
- 대안: 미응답 문항을 0점 처리.
  - 미채택 이유: 핵심 4문항만 응답한 사용자에게 과도하게 낮은 점수를 주는 편향이 발생한다.

### 4) 우선 과제 추천은 미실행 + 사업 임팩트 가중치로 계산한다
- 선택: 질문별 `impact_weight`를 정의하고, 우선순위를 `(2-score) * impact_weight`로 계산해 상위 1개를 추천한다.
- 이유: “안 하고 있는 항목”이 많을 때도 사업에 미치는 효과가 큰 과제가 먼저 나오도록 보장할 수 있다.
- 대안: 단순 저점수(0점 우선) 정렬.
  - 미채택 이유: 실행 난이도/사업 가치가 낮은 항목이 과대 추천될 수 있다.

### 5) 시뮬레이션/이메일은 공통 섹션 계약을 유지하고 문항 의도 키로 매칭한다
- 선택: 공통 섹션 AST를 유지하고, 카테고리 메시지/우선과제 문구는 질문 의도 키(축 + 행동 타입) 기반으로 매핑한다.
- 이유: 결과 화면/시뮬레이션/이메일 간 문구 드리프트를 줄이고 QA 포인트를 단일화할 수 있다.
- 대안: 채널별 별도 카피 조정.
  - 미채택 이유: 단기 수정은 쉬우나 장기적으로 메시지 불일치 회귀가 반복된다.

## Risks / Trade-offs

- [선택 문항 응답률이 낮아 진단 정밀도가 떨어질 수 있음] -> 결과에 간단/정밀 상태를 명시하고, 추가 문항 응답 유도 문구를 제공한다.
- [정규화 기준 변경으로 기존 점수 체감이 달라질 수 있음] -> 스냅샷/회귀 테스트를 갱신하고 임계값 변경 여부를 검증한다.
- [임팩트 가중치 설계가 주관적으로 보일 수 있음] -> 가중치 기준(매출/시간절감/운영리스크)을 문서화하고 변경 지점을 고정한다.

## Migration Plan

1. `ax_tool_stack`에 질문 메타(핵심/선택, 임팩트 가중치, 표시 순서)를 추가한다.
2. 설문 템플릿을 카테고리 블록 방식에서 순차 질문 + 선택 확장 방식으로 변경한다.
3. 점수 계산/등급 계산을 응답 기반 정규화 방식으로 조정한다.
4. 우선 과제 추천 함수를 임팩트 가중치 기반으로 개편한다.
5. 결과 시뮬레이션/이메일 카피 매핑을 갱신하고 공통 섹션 출력 동등성을 검증한다.
6. 테스트(`test_contact_form`, `test_landing_pages`, `test_lead_magnet_sections`) 기대값을 업데이트한다.
